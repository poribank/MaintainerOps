# Code of Conduct

MaintainerOps is for maintainers and contributors who need reliable, reviewable open source operations. Project spaces should stay focused, respectful, and safe for people doing maintenance work.

## Expected Behavior

- Be direct and constructive when discussing bugs, risk, security, and design tradeoffs.
- Respect maintainer time by keeping issues and pull requests scoped and reproducible.
- Assume repository access, security findings, and AI-related data handling are sensitive unless clearly public.
- Disclose conflicts of interest when proposing automation that changes repository permissions, release flows, or data retention.

## Unacceptable Behavior

- Harassment, personal attacks, discriminatory language, or sustained disruption.
- Publishing private repository data, webhook payloads, secrets, installation tokens, or maintainer contact details.
- Using MaintainerOps workflows to probe repositories without owner, maintainer, or administrator authorization.
- Pressuring maintainers to bypass review, branch protection, security policy, or explicit approval requirements.

## Scope

This Code of Conduct applies in MaintainerOps project spaces, including the repository, issues, pull requests, security reports, release discussions, and official project communication channels.

## Reporting

For security-sensitive reports, use GitHub private vulnerability reporting:

https://github.com/poribank/MaintainerOps/security/advisories/new

For non-security conduct concerns, contact the maintainers listed in [MAINTAINERS.md](MAINTAINERS.md). Include relevant links, dates, and context. Do not include private data that is not needed to evaluate the report.

## Enforcement

Maintainers may remove comments, close discussions, request behavior changes, restrict participation, or escalate to GitHub platform moderation when needed. Enforcement should be proportionate, documented where appropriate, and focused on protecting the project and its contributors.
