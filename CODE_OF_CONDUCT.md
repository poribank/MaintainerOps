# Code of Conduct

MaintainerOps is an open source maintainer tool, and project spaces should stay professional, respectful, and useful for contributors with different backgrounds and levels of experience.

## Expected behavior

- Be respectful and constructive in issues, pull requests, reviews, and discussions.
- Assume good intent, but accept correction when your impact is harmful.
- Keep feedback focused on the work, evidence, and maintainability.
- Respect privacy and avoid posting secrets, private repository content, private messages, or personal information.
- Use private vulnerability reporting for suspected security issues instead of public disclosure.

## Unacceptable behavior

- Harassment, threats, intimidation, stalking, or sustained disruptive behavior.
- Insults, slurs, sexualized language or imagery, or unwelcome attention.
- Publishing private information without explicit permission.
- Deliberately derailing maintainer workflows, security response, or release processes.
- Retaliation against anyone who raises a conduct or security concern in good faith.

## Scope

This Code of Conduct applies in MaintainerOps project spaces, including the repository, issues, pull requests, security reports, release discussions, and any official project communication channels.

## Reporting

For security-sensitive reports, use GitHub private vulnerability reporting:

https://github.com/poribank/MaintainerOps/security/advisories/new

For non-security conduct concerns, contact the maintainers listed in [GOVERNANCE.md](GOVERNANCE.md). Include relevant links, dates, and context. Do not include private data that is not needed to evaluate the report.

## Enforcement

Maintainers may remove comments, close discussions, request behavior changes, restrict participation, or escalate to GitHub platform moderation when needed. Enforcement should be proportionate, documented where appropriate, and focused on protecting the project community and maintainership capacity.

## Attribution

This policy is informed by common open source community standards, including the Contributor Covenant, version 2.1.
