# Code of Conduct

MaintainerOps is for maintainers and contributors who need reliable, reviewable open source operations.
Project spaces should stay focused, respectful, and safe for people doing maintenance work.

## Expected Behavior

- Be direct and constructive when discussing bugs, risk, security, and design tradeoffs.
- Respect maintainer time by keeping issues and pull requests scoped and reproducible.
- Assume repository access, security findings, and AI-related data handling are sensitive unless clearly public.
- Disclose conflicts of interest when proposing automation that changes repository permissions, release flows, or data retention.

## Unacceptable Behavior

- Harassment, personal attacks, discriminatory language, or sustained disruption.
- Publishing private repository data, webhook payloads, secrets, installation tokens, or maintainer contact details.
- Using MaintainerOps workflows to probe repositories without owner, maintainer, or administrator authorization.
- Pressuring maintainers to bypass review, branch protection, security policy, or explicit approval requirements.

## Reporting

For security vulnerabilities, use GitHub private vulnerability reporting:

https://github.com/poribank/MaintainerOps/security/advisories/new

For conduct concerns that are not security vulnerabilities, contact the maintainers listed in
[GOVERNANCE.md](GOVERNANCE.md). Maintainers may remove content, pause discussions, block abusive accounts, or
limit participation when needed to protect the project and its contributors.
